PART 1

캔바 AI로
웹페이지 만들기

가입 → 말로 주문 → 게시. 브라우저 하나면 끝납니다.

PART 1 · 준비

먼저 교사 인증부터 받으세요

교사 인증을 하면

  • 유료 기능(캔바 Pro)을 무료로 사용
  • AI 기능 사용량이 넉넉해집니다
  • 학급/수업 공유 기능 사용 가능

인증 방법

  • canva.com/education 접속
  • 학교 이메일(@…sen.go.kr 등) 또는 교원증·재직증명서 제출
  • 보통 1~2일 내 승인
TIP 연수 시작 일주일 전에 참가 선생님들께 인증 신청을 미리 안내하세요. 당일에 하면 실습을 못 합니다.
PART 1 · 실습

따라 하기 · 5단계

1
Canva AI 열기로그인 → 홈 화면에서 Canva AI 선택 → 만들고 싶은 것을 한국어로 그냥 설명합니다.
2
프롬프트 입력예) "초등 5학년 사회 '우리 지역 문화유산' 학습용 웹페이지를 만들어줘. 사진 4장, 설명 카드, 마지막에 3문항 퀴즈."
3
코드 생성 기능 활용단순 페이지가 아니라 동작하는 프로그램(퀴즈·타이머·뽑기)이 필요하면 코드 생성을 선택합니다.
4
수정 요청 반복"글씨 더 크게", "정답 맞히면 축하 효과" 처럼 말로 고칩니다. 마음에 들 때까지 반복하세요.
5
웹사이트로 게시우측 상단 공유 → 웹사이트 게시 → 주소 입력 → 완성된 링크를 학생에게 전달.
PART 1 · 정리

캔바의 장점과 한계

✓ 좋은 점

  • 설치·가입 외에 배울 것이 거의 없음
  • 디자인이 기본으로 예쁩니다
  • 교육용 계정이 무료 — 예산 결재 불필요
  • 수정이 즉시 반영되어 반응이 빠릅니다

✗ 한계

  • CSV·GIF·mp3 등 자료를 한 개씩 올려야 함
  • 웹사이트 주소는 한 번 정하면 변경이 번거로움
  • 캔바 밖으로 코드를 가져가기 어려움
  • 외부 데이터와 연결하는 데 제약
그래서 다음 단계로 자료가 많아지는 순간 ② 제미나이+GitHub 방식이 훨씬 편해집니다.

PART 2 · 개념

역할이 둘로 나뉩니다

① 제미나이 Canvas = 만드는 곳

  • AI와 대화하며 코드를 생성
  • 화면 오른쪽에 실시간 미리보기
  • 여기서는 아직 나만 볼 수 있음

② GitHub Pages = 올리는 곳

  • 만든 파일을 인터넷에 공개
  • 무료 · 주소가 영구히 유지됨
  • 자료 폴더(이미지·PDF·mp3)를 함께 보관
기억할 한 줄 제미나이는 주방, GitHub Pages는 식당입니다. 만드는 곳과 내놓는 곳이 다릅니다.

PART 2 · 주의

반드시 지켜야 할 3가지

!
파일 이름은 index.html저장소 최상단(루트)에 index.html이 없으면 404 화면만 나옵니다. 가장 흔한 실수입니다.

실제 사고가 가장 많은 지점입니다. 명단이 들어간 엑셀을 실습 자료로 쓰다가 그대로 올리는 경우가 있습니다. 연수 실습 자료는 가상 이름으로 만들어 배포하세요.

구글 시트 + 앱스크립트로
웹페이지 만들기

학생이 입력한 내용이 시트에 쌓이는 웹페이지. 우리나라 학교에서 가장 널리 쓰이는 방식입니다.

PART 3 · 개념

구글 시트를 데이터베이스로 씁니다

Code.gs — 뒤에서 일하는 부분

  • 시트에서 데이터를 읽어오고
  • 웹에서 들어온 값을 시트에 기록
  • doGet() / doPost() 두 함수가 핵심

Index.html — 학생이 보는 화면

  • 버튼·입력창·목록 등 화면 전체
  • 평범한 HTML이라 AI가 잘 만들어 줍니다
비유하자면 시트는 창고, Code.gs는 창고 직원, Index.html은 손님이 보는 매장입니다.
PART 3 · 실습

따라 하기 · 6단계

1
구글 시트 만들기1행에 항목 이름(예: 시각 / 이름 / 답변)을 적어 둡니다. 이게 데이터 설계의 전부입니다.
2
확장 프로그램 → Apps Script시트 상단 메뉴에서 열면 코드 편집기가 새 탭으로 뜹니다.
3
Code.gs 붙여넣기다음 장의 코드를 그대로 붙여넣습니다. (AI에게 "이 시트를 읽는 doGet 만들어줘"라고 시켜도 됩니다)
4
Index.html 추가왼쪽 파일 + → HTML → 이름을 Index 로. 화면 디자인은 AI에게 맡기세요.
5
배포우측 상단 배포 → 새 배포 → 유형: 웹 앱 → 실행 계정·액세스 권한 설정 → 배포.
6
주소 공유발급된 웹 앱 URL을 학생에게 전달. 코드를 고칠 때마다 새 배포를 해야 반영됩니다.
PART 3 · 코드

Code.gs — 이만큼이면 동작합니다

// ① 웹 주소로 접속하면 Index.html 화면을 보여준다
function doGet() {
  return HtmlService.createTemplateFromFile('Index').evaluate();
}

// ② 시트에 있는 데이터를 전부 읽어서 화면으로 넘겨준다
function getData() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('시트1');
  return sheet.getDataRange().getValues();
}

// ③ 화면에서 보낸 값을 시트 맨 아래에 한 줄로 추가한다
function addRow(name, answer) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('시트1');
  sheet.appendRow([new Date(), name, answer]);
  return '저장 완료';
}

화면(Index.html)에서는 google.script.run.getData() 처럼 함수 이름을 불러 쓰면 됩니다.

마무리 2 · 가장 중요

공교육에서 꼭 확인할 것

  • 기본값은 "수집하지 않기". 이름 없이 번호만, 또는 아예 저장하지 않는 설계로도 대부분의 수업 도구는 충분합니다.
  • 학생 개인정보를 수집하면 심의 대상입니다. 초·중등교육법 제29조의2에 따라, 교사가 직접 만든 소프트웨어라도 학교운영위원회 심의를 거쳐야 합니다.
  • 필수 기준 — 최소 수집 / 안전 조치 / 열람·정정·삭제 요구 처리 / 만 14세 미만 아동 보호. 하나라도 못 지키면 사용할 수 없습니다.
  • 공개 저장소에 명단 금지. GitHub은 삭제해도 기록이 남습니다.
  • 비밀번호를 코드에 적지 않기. 로그인이 필요하면 학교 구글 계정 인증을 사용하세요.
